# 要件定義書

## ラポール7110 Webサイトリニューアル

---

## 1. プロジェクト概要

### 1.1 基本情報

| 項目 | 内容 |
|------|------|
| プロジェクト名 | ラポール7110 Webサイトリニューアル |
| クライアント | ラポール7110（就労継続支援施設） |
| 種別 | リニューアル（新規ではない） |
| 目標期間 | 1〜2ヶ月 |
| 予算目安 | 20万円＋α（写真撮影費別途） |

### 1.2 プロジェクトの目的

1. 施設の特色・強みを明確に伝える
2. 利用検討者（本人・親御さん）の不安を解消する
3. 問い合わせ前の期待値調整を行う
4. 広告運用との連携を見据えた情報設計

### 1.3 背景

- 今後、広告を回して集客を強化する予定
- 問い合わせ増加に伴い「サイトを見てください」で解決できる状態を目指す
- 来訪後のミスマッチ（「先に言ってほしかった」等）を減らしたい

---

## 2. ターゲットユーザー

### 2.1 メインターゲット

| ターゲット | 特徴・ニーズ |
|-----------|-------------|
| ご本人（利用者候補） | 作業内容、雰囲気、通いやすさを知りたい |
| 親御さん | 安心して預けられるか、サポート体制を確認したい |

### 2.2 サブターゲット

| ターゲット | 特徴・ニーズ |
|-----------|-------------|
| 特別支援学校 | 生徒の進路先として検討 |
| 周辺福祉施設 | 連携・紹介先として情報収集 |

### 2.3 ユーザー行動の想定

- 比較検討するユーザーが多い
- サイト内をしっかり回遊して情報収集する
- LP的な縦長1ページではなく、複数ページを見て判断

---

## 3. サイト設計方針

### 3.1 基本方針

| 方針 | 説明 |
|------|------|
| 回遊型設計 | LP設計ではなく、複数ページを見渡してもらう構成 |
| 情報充実 | 比較検討ユーザー向けに情報は多めに掲載 |
| 不安解消 | ネガティブ要素も事前に伝えて期待値調整 |
| 既存ベース | デザインは既存を活かしつつ拡充 |

### 3.2 重要なコンテンツ要素

#### 必須コンテンツ
- [ ] 作業内容の詳細（農園、梱包、等）
- [ ] 1日のスケジュール・タイムライン
- [ ] 具体的な作業シーン（写真）
- [ ] 施設へのアクセス方法（入り方の案内）
- [ ] 施設の特色・強み

#### 事前に伝えるべき情報（期待値調整）
- [ ] 駅からの距離（遠い）
- [ ] 施設の場所（2階、建物の奥）
- [ ] 階段利用について

## 6. デザイン要件

### 6.1 基本方針

- 既存デザインをベースに拡充
- 大幅なデザイン変更は行わない
- 写真を活用して実態を伝える

### 6.2 写真撮影

| 撮影対象 | 優先度 |
|----------|--------|
| 作業シーン | 高 |
| 施設内観 | 高 |
| スタッフ | 中 |
| 外観・入口 | 中 |
