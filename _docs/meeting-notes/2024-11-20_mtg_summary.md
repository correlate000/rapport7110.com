## 議題・決定事項

### 1. プロジェクト概要

- **目的**: 既存Webサイトのブラッシュアップ・リニューアル
- **期間目標**: 1〜2ヶ月程度での着地を目指す
- **予算感**: 20万円＋α程度（写真撮影費は別途）
- **方針**: システム面よりも載せる内容を固めることを優先

### 2. ターゲット

| 優先度 | ターゲット |
|--------|-----------|
| メイン | ご本人（利用者）、親御さん |
| サブ | 特別支援学校、周辺施設 |

- LP設計ではなく、**回遊してサイト内を見渡してもらう設計**が適切
- 比較検討するユーザーが多いため、情報は多く載せて良い

### 3. 現状の課題・改善ポイント

#### ネガティブ要素（事前に伝えるべき情報）
- 駅から遠い
- 2階にあり階段を使う
- 建物の1番奥にある（入り方の案内が必要）

#### コンテンツの課題
- 作業内容の具体性が不足
- 施設の特色・強みが伝わりにくい
- 写真がフリー素材中心で実態が伝わらない

### 4. 掲載したい情報・コンテンツ

| カテゴリ | 内容 |
|----------|------|
| 作業内容 | 農園、梱包、その他の作業詳細 |
| 作業スケジュール | 1日の流れ、具体的な作業シーン |
| 施設情報 | アクセス方法（入り方の案内含む） |
| 施設の特色 | 強み・売りになる部分 |
| ニュースリリース | 既存の仕組みを継続 |

### 5. 写真撮影について

- 作業シーンの撮影が必要
- 施設写真の更新
- 既存写真（模様替え記事等）も活用可能
- 撮影時期：構成確定後、並行して進める可能性あり

### 6. 技術面

- **CMS**: WordPress（継続）
- **サーバー**: Xサーバー（クライアント自社管理）
- **対応**: サーバー情報の共有を依頼

## 進め方・次のステップ

### Phase 1: 情報設計
1. 載せたい情報の整理
2. ページ構成案の作成
3. トップページの要素検討（何を置く・外す）

### Phase 2: デザイン・構築
- 既存デザインベースで対応
- ページの拡充
- デザインと構築はリソース確保できれば比較的早い

### Phase 3: 写真撮影
- 構成確定と並行して進行可能
